Лог изменений интеграции 1С

Дата изменений: 30 марта 2026. Проект: avterminal-api.

amoCRM → 1С debug endpoints payload fix

1) Что исправили в формате payload для 1С

Проблема: в ряде случаев в очереди для 1С уходил clientType=legal, но компания была пустой:

  • client.name = null
  • client.inn = ""
  • _meta.companyId = null

Причина: при сборке payload информация о компании не всегда подтягивалась из API-связи сделки.

Исправление: добавлен fallback на входящий webhook payload от amoCRM (lead/contact/company), включая поддержку legacy-структуры custom_fields.

2) Два тестовых метода (без авторизации)

Оба метода доступны без auth:sanctum, как и /api/amocrm/logs.

2.1 Последние статусы очереди

GET /api/amocrm/integrations/1c/debug/statuses?limit=5

  • Возвращает последние записи таблицы onec_counterparty_buffers (по умолчанию 5, максимум 50).
  • Для каждой записи добавляет готовую ссылку pullByRequestUrl на второй debug-метод.
{
  "count": 5,
  "items": [
    {
      "requestId": "req_jrivmeh6gwib",
      "status": "pending",
      "pullByRequestUrl": "http://api-avto-terminal.ru/api/amocrm/integrations/1c/debug/pull-by-request?requestId=req_jrivmeh6gwib"
    }
  ]
}

2.2 Pull по requestId (показать, что реально уходит в 1С)

GET /api/amocrm/integrations/1c/debug/pull-by-request?requestId=...

  • Возвращает фактический payload, который выдается 1С.
  • Если запись была в pending, метод переводит ее в pulled, увеличивает pullAttempts и пишет pulled-event.
  • Если запись уже не pending, просто возвращает текущий статус и payload.
{
  "requestId": "req_jrivmeh6gwib",
  "previousStatus": "pending",
  "currentStatus": "pulled",
  "wasPulledNow": true,
  "pullAttempts": 1,
  "payload": {
    "...": "фактические данные для 1С"
  }
}

3) Быстрые ссылки для проверки

  • GET debug/statuses?limit=5
  • GET /api/amocrm/integrations/1c/debug/pull-by-request?requestId=REQ_ID
  • GET amocrm/logs?limit=20

Примечание: pull-by-request меняет состояние очереди для pending-записей (это диагностический pull, а не read-only просмотр).
