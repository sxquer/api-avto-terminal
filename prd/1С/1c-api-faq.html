FAQ для программиста 1С по API

Базовый URL: http://api-avto-terminal.ru

1) /api/amocrm/integrations/1c/contacts/result: можно передавать массив или только по одному?

Сейчас только по одному.

Endpoint валидирует одиночный объект с обязательным requestId и status. Массив объектов не поддержан.

{
  "requestId": "req_xxxxxxxxxxxx",
  "status": "created | found | error",
  "1cId": "optional",
  "vin": "optional",
  "processedAt": "optional",
  "error": "optional"
}

2) /api/amocrm/integrations/1c/contacts/pending?limit=50: как работает limit?

Да, limit работает.

  • Параметр limit необязательный.
  • Если не передать, по умолчанию используется 50.
  • Допустимый диапазон: 1..200.

Каждый запрос к pending возвращает новую порцию из записей со статусом pending, и сразу помечает их как pulled. Поэтому повторный вызов не вернет уже выданные записи, если их не перевести обратно в pending.

3) Пример header для запросов 1С

GET pending:

Authorization: Bearer <SANCTUM_TOKEN>
Accept: application/json

POST result:

Authorization: Bearer <SANCTUM_TOKEN>
Accept: application/json
Content-Type: application/json

4) processedAt: что отправлять?

Рекомендуется передавать дату/время фактической обработки в 1С в формате ISO-8601 с timezone, например:

"processedAt": "2026-03-20T14:25:00+10:00"

Если поле не передать, API поставит текущее время сервера автоматически.

5) Минимальные рабочие примеры

Получить pending:

GET http://api-avto-terminal.ru/api/amocrm/integrations/1c/contacts/pending?limit=50

Отправить результат:

POST http://api-avto-terminal.ru/api/amocrm/integrations/1c/contacts/result

{
  "requestId": "req_jvqa7tml46vk",
  "status": "created",
  "1cId": "1c-ctr-legal-0003",
  "processedAt": "2026-03-20T14:25:00+10:00"
}
