Схема проверки дублей и очереди (amoCRM → 1С)

Актуально для endpoint постановки в буфер контрагентов после обновления логики дедупликации.

1. Ключевой принцип

Дедупликация выполняется только по payload_hash в рамках одной сделки (lead_id).

  • Если по этой сделке уже есть запись с таким же payload_hash, новая запись не создается.
  • Если payload_hash изменился, создается новая запись в очереди, даже если старая еще pending или pulled.

2. Как считается payload_hash

payload_hash = sha256(json(payload)), где payload формируется из бизнес-данных карточки:

  • vin
  • dealId
  • clientType, clientTypeRaw
  • client (данные физ/юрлица)
  • deal
  • _meta

Важно: requestId в payload добавляется после расчета хэша, поэтому сам по себе новый requestId не влияет на dedupe.

3. Поведение очереди

Ситуация Результат
Первая отправка карточки Создается запись pending, выдается новый requestId.
Повторная отправка без изменений Новая запись не создается, возвращается already_buffered и requestId существующей записи с тем же hash.
Карточка изменена, старая запись еще pending/pulled Создается новая запись (новый hash, новый requestId). Старая запись остается в очереди.
Карточка изменена после processed или error Создается новая запись (если hash новый).

4. Жизненный цикл записи

  1. pending: запись создана и ждет, когда 1С заберет ее через GET /pending.
  2. pulled: запись уже отдана 1С.
  3. processed или error: 1С прислала callback в POST /result.

5. Почему это удобно для бизнеса

  • Можно исправить данные в amoCRM и сразу отправить новую версию карточки.
  • Текущий цикл обработки в 1С не ломается: старая запись остается валидной до своего callback.
  • Защита от дублей сохраняется: одинаковая версия payload не попадет в очередь повторно.

6. Короткий пример

t0: отправка #1 (hash=A) => pending, requestId=req_1
t1: оператор меняет ИНН в карточке
t2: отправка #2 (hash=B) => pending, requestId=req_2 (создана новая запись)
t3: 1С обрабатывает req_1 и затем req_2
t4: итоговое состояние контрагента соответствует последней версии данных (hash=B)
