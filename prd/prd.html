Техническое задание: интеграция AmoCRM – Альта

В рамках интеграции АмоСРМ и Альта ГТД планируется решить 2 задачи:
1) Автоматизация заполнения пассажирской таможенной декларации (ПТД) в Альта-Софт, путем разработки виджета в АмоСРМ, формирующего XML файл, который сотрудники будут выгружать вручную для переноса в АльтаГТД.

2) Автоматизация передачи уведомлений о статусах ПТД (регистрация, выпуск, отказ) в AmoCRM. Альта ГТД отдает – АмоСРМ принимает.
Тут со стороны Амо нужно будет:
- формат данных согласовать на вход
- настроить обработку входящей информации со стороны АМО.

3) Автоматизация передачи уведомлений о статусах транзитных ДТ (далее – ДТ) (регистрация, … другие статусы уточняются) в AmoCRM. Альта ГТД отдает – АмоСРМ принимает.

1.Автоматизация заполнения пассажирской таможенной декларации (ПТД) в Альта-Софт.

Необходимо разработать функционал для выгрузки данных из AmoCRM (виджет) в формате XML для последующего ручного импорта сотрудником в ПС Альта-ГТД.


К этой части ТЗ прилагается:
- xml файл, выгруженный из Альта ГТД, как есть – может использоваться для анализа структуры, узлов и т.д.

- пример заполнения xml файла из данных Амо-СРМ. Xml содержит минимально-необходимое кол-во полей.
Он загружается в Альта-ГТД и вся информация встает на свои места.
Послодовательность узлов не в полной мере соответствует исходинку, однако для xml файла это не критично. Главное наличие самих узлов и внутренней структуры внутри блоков.

В Карточке АМО СРМ необходимо разделить ФИО на отдельные ячейки.
Сейчас данные по ФИО физ лица, привязанного к карточке сделки, содержится в одном поле.
Задача разделить на 3.
При этом, как к  экспертам в области структуры АМО СРМ, возникает вопрос, как лучше сделать с учетом того, что ФИО физика уже используется в печатных формах.
Видится, что целесообразно сделать так:
в карточке физ.лица ввести дополнительные 3 поля:
Фамилия, Имя, Отчетство.
А текущее поле, которое в системе называется «Фамилия» сделать как авоматическую сцепку из созданных полей.
В этом случае не будет изменена логика привязки полей к текущим печатным формам и другим связям в системе, которые есть, но появится возможность использовать разделенные Фамилию, Имя, Отчетство для формирования xml, без использования сторонних сервисов (в КП предложено использование DaData).
